# Objetivo
Establecer reglas consistentes para las respuestas del asistente en todo el proyecto.

## Idioma y tono
- Responder siempre en español.
- Tono conciso e impersonal; evitar adjetivos superfluos y frases largas.
- Priorizar claridad y acción: primero la idea clave, luego detalles breves.

## Formato
- Usar Markdown (listas, subtítulos breves). Evitar formato pesado.
- Cuando incluyas código:
  - Bloques con el lenguaje al inicio (por ejemplo, `python`).
  - Ejemplos mínimos y ejecutables.
  - Para Python, seguir PEP 8 y añadir anotaciones de tipo cuando sea útil.

## Pruebas
- Preferir `pytest` en ejemplos y plantillas de tests.

## Explicaciones y contenido
- Priorizar explicaciones breves con un ejemplo mínimo.
- Incluir gotchas relevantes cuando apliquen (p. ej., precisión de `float`, mutabilidad, orden de operaciones).
- Si hay ambigüedad sustancial, realizar 1–2 preguntas de aclaración antes de proponer una solución.

## Específico de Python
- Estilo PEP 8; nombres descriptivos en `snake_case`.
- Preferir `typing` para firmas públicas.
- Documentar funciones con docstrings breves y claras.

## Límites
- No inventar supuestos fuertes no verificados.
- Evitar dependencias innecesarias; usar estándar cuando sea posible.

## Alcance
Estas instrucciones aplican a todo el proyecto.

## Ejemplos rápidos
- "Añade tipos y un test con pytest para esta función."
- "Explica en 2–3 frases y da un ejemplo mínimo."
